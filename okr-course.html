OObjective（目标）—— 定方向

Objective（目标）回答的是：你要达成什么？它是定性的、鼓舞人心的方向，不是冷冰冰的数字。

❌ 不好
"提高销售额"
✅ 好
"成为客户首选的供应商"
自检提示：这个目标说出来，团队成员会觉得兴奋吗？如果听起来像 KPI 考核表，就不是好的 Objective（目标）。
随堂练习

哪个更像一个好的 Objective（目标）？

A 季度营收增长 30%（偏 KR，偏定量）
B 我们要成为行业最值得信赖和认可的品牌
C 完成三次产品迭代（偏 KR，偏任务）
✅ 正确！B 是定性描述、鼓舞人心，能让团队成员感受到方向感。A 和 C 都是数字或任务，更适合做 Key Results（关键结果），而不是 Objective（目标）。
❌ 再想想。Objective（目标）的核心是"定方向、鼓舞人"，不是写数字。A 是定量指标（像 KPI），C 是任务清单。B 是定性描述，说出了"我们想成为什么"，这才是好的 Objective（目标）。

KKey Results（关键结果）—— 定指标

Key Results（关键结果）回答的是：你怎么证明达成了 Objective（目标）？它是定量的、可衡量的，有明确的完成标准。

一个 Objective（目标）通常配 3-5 个 Key Results（关键结果）。

❌ 不可衡量
"优化用户体验"
✅ 可衡量
"NPS（净推荐值）评分从 35 提升至 50"
自检提示：当 KR 完成时，能不能明确说"达成了"或"没达成"？如果不能精确衡量，就不是好的 Key Results（关键结果）。
随堂练习

哪个是好的 Key Results（关键结果）？

A 优化用户体验（不可衡量）
B NPS（净推荐值）评分从 35 提升至 50（可衡量、有基准）
C 努力工作（不可衡量）
✅ 正确！B 有明确的衡量指标（NPS 评分）、基准值（35）和目标值（50），是典型的 Key Results（关键结果）。A 和 C 没有数字或标准，无法判断完成与否。
❌ 再想想。Key Results（关键结果）的核心是"可衡量"。A 和 C 都没有数字或标准，无法判断完成与否。B 有"从 35 到 50"这个明确的变化，而且有具体指标名称（NPS 净推荐值），最符合标准。

好的 Objective（目标）长什么样？

一个优秀的 Objective（目标）必须满足三个标准：

  • 1. 鼓舞人心（Inspiring）— 不是干巴巴的数字，能让团队成员感到兴奋
  • 2. 定性描述（Qualitative）— 不说具体数字，用方向性语言描述"我们要成为什么"
  • 3. 简洁好记（Concise）— 一句话能说清楚，团队每个人都能脱口而出
❌ 不好
"Q3 销售额提升 20%"
✅ 好
"让客户主动推荐我们的产品"
记忆口诀：好的 Objective（目标） = 让人兴奋的方向 + 不含数字 + 一句话能说清。
随堂练习

以下哪个 Objective（目标）最符合"鼓舞人心+定性"标准？

A 本季度签下 50 家新客户（含数字，偏 KPI）
B 成为客户愿意主动推荐的首选品牌
C 完成 CRM 系统上线（偏任务，不含方向感）
✅ 正确！B 是定性描述，没有具体数字，而且"首选品牌""主动推荐"都有鼓舞人心的方向感。A 含数字，更适合做 KR；C 是任务清单，没有战略方向。
❌ 再想想。好的 Objective（目标）要"鼓舞人心+定性"。A 含数字（50 家），更像 KPI 考核指标；C 是任务（完成上线），没有方向感。B 是"成为什么"，没有数字，是鼓舞人心的方向描述。

好的 Key Results（关键结果）长什么样？

一个优秀的 Key Results（关键结果）必须满足三个标准：

  • 1. 可衡量（Measurable）— 有数字或明确的评判标准，完成就是完成，没完成就是没完成
  • 2. 有挑战性（Ambitious）— 完成率 60%-70% 是理想区间，不是 100% 才算好。如果每次都 100% 完成，说明目标不够有挑战
  • 3. 时限清晰（Time-bound）— 有明确的截止日期，不能无限期拖延
❌ 不好
"提升用户活跃度"
✅ 好
"日活跃用户从 2000 提升至 3500（Q2 末）"
核心原则：Key Results（关键结果）不怕完不成，怕的是不够有挑战。完成 70% 的 OKR 比完成 100% 的平庸 KPI 更有价值。
随堂练习

以下哪个 Key Results（关键结果）的问题最大？

A 网站访问量在 3 个月内从 1 万增至 3 万（可衡量、有时限）
B 客户满意度提升 10%（可衡量，但缺时限）
C 提升团队战斗力（不可测、无时限、无基准）
✅ 正确！C 的问题最大——不可衡量、没有时限、也没有基准值，完全无法判断完成与否。A 虽然挑战大，但结构完整。B 缺时限，但比 C 好得多。
❌ 再想想。题目问的是"问题最大"的那个。A 虽然挑战性大（3 倍增长），但结构完整（有数字、有基准、有时限）。B 缺时限但可衡量。C 三重缺失：不可测、无时限、无基准，问题最大。

OKR（目标与关键结果） vs KPI（关键绩效指标）

这两个工具经常被混淆，但本质完全不同：

  • KPI（关键绩效指标）：考核工具，"必须达成的指标"。完不成 = 绩效差，可能影响奖金和晋升
  • OKR（目标与关键结果）：对齐工具，"争取达成的方向"。KR 完成 70% 就是优秀，重点是方向一致和过程透明
KPI 思维
销售额必须达成 1000 万（完不成扣绩效）
OKR 思维
销售额争取 1500 万（即使达成 1000 万，也有明确进展）
核心区别：OKR（目标与关键结果）不怕完不成，怕的是目标不够有挑战。KPI（关键绩效指标）是"底线"，OKR 是"上限"。
随堂练习

以下场景更适合用 OKR（目标与关键结果）还是 KPI（关键绩效指标）？

场景：团队本季度要达成的目标是"开发一个创新功能原型（但不一定能保证上线）"

A 用 KPI（关键绩效指标）考核（必须上线）
B 用 OKR（目标与关键结果）管理（探索方向，允许试错）
C 两个都用，效果一样
✅ 正确！创新探索类任务不适合用 KPI（关键绩效指标）硬考核——"不一定上线"本身就说明有试错空间。OKR（目标与关键结果）更适合这类探索方向、允许试错、需要团队对齐的目标。C 也不对，因为 KPI 和 OKR 的适用场景完全不同。
❌ 再想想。这个场景明确说了"不一定能保证上线"，说明是探索性、创新性的任务。KPI（关键绩效指标）是"必须达成"的考核工具，不适合这种不确定的任务。OKR（目标与关键结果）允许试错，关注的是方向对齐和过程透明，而不是 100% 完成率。

OKR（目标与关键结果）设定的重要原则

写好 OKR（目标与关键结果），除了掌握 O 和 KR 的区别，还要记住三个核心原则：

  • 1. 聚焦（Focus）：每人/每团队每季度 1-3 个 O（目标），不要超过 3 个。写太多等于没有重点，执行必然打折扣
  • 2. 对齐（Align）：OKR（目标与关键结果）是"自下而上提出 + 自上而下对齐"。个人提出的目标要和团队/公司方向一致，不能各干各的
  • 3. 透明（Transparent）：所有人的 OKR（目标与关键结果）公开可见。减少信息不对称，让协作更顺畅
❌ 不聚焦
一个人写 8 个 O（目标）、20 个 KR（关键结果）
✅ 聚焦
一个人写 2 个 O（目标）、5 个 KR（关键结果）
核心原则：OKR（目标与关键结果）不是"越多越好"，而是"越聚焦越有效"。1-3 个 O（目标）足以定义一个季度的战略方向。
随堂练习

一个团队定了 5 个 Objective（目标）、每个配 5 个 Key Results（关键结果），问题是什么？

A 写得太少，没有覆盖所有工作
B 写得太多了，失去聚焦，执行必然打折扣
C 数量没问题，只要时间够
✅ 正确！5 个 O（目标）× 5 个 KR（关键结果）= 25 个指标，完全失去了聚焦。OKR（目标与关键结果）的核心原则是"每季度 1-3 个 O（目标）"，数量越少、越聚焦，执行效果越好。
❌ 再想想。OKR（目标与关键结果）的核心原则就是"聚焦"。每人/每团队每季度 1-3 个 O（目标），超过 3 个就失去重点。25 个 KR（关键结果）分散精力，执行必然打折扣。OKR 不是"覆盖所有工作"，而是"聚焦最重要的方向"。
